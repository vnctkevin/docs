  # Create Transactions

# [Overview](#overview)

Every transaction on the Fireblocks platform is initiated using the <a target="_self" href="/reference/createtransaction">Create a new transaction</a> API call. Its default operation type is <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">TRANSFER</code>, however, other important operations are available as valid values for the <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">operation</code> body parameter.

<a target="_self" href="/reference/data-objects#transactionoperation">See all <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">operation</code> valid values.</a>

The main available transaction operations are:

*   TRANSFER: transfer an asset from one source to one destination
*   CONTRACT\_CALL: Directly invoke a smart contract
*   RAW: Sign arbitrary data using the Raw Signing API
*   MINT: Perform a mint operation (increase supply) on a supported token
*   BURN: Perform a burn operation (reduce supply) on a supported token
*   TYPED\_MESSAGE: allows users to sign messages, but only using specific standard formats that prefix the message with a magic string, such as [Ethereum personal messages](https://geth.ethereum.org/docs/rpc/ns-personal#personal_sign), and (EIP712 typed structured data) type structured data

The API call serves all possible source and destination combinations including a one-time address destination (OTA), vault account, pre-whitelisted internal wallet, external wallet & contract wallet, Fireblocks Network connections and any connected exchange account and fiat account.

# [Your first transaction](#your-first-transaction)
## [Vault account to vault account transaction](#vault-account-to-vault-account-transaction)

To start, initiate a transaction from one Fireblocks vault account to another. Account-based assets allow single-destination transfer and UTXO assets allow multi-destination transfer.

### [Single destination transfer (ETH)](#single-destination-transfer-eth)

The example transfers 0.001 amount of ETH from vault account ID <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">0</code> to vault account ID <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">1</code>.

<CodeGroup>
```typescript
const transactionPayload = {
  assetId: "ETH",
  amount: "0.001",
  source: {
    type: TransferPeerPathType.VaultAccount,
    id: "0",
  },
  destination: {
    type: TransferPeerPathType.VaultAccount,
    id: "1",

  note: "Your first transaction!",

createTransaction(transactionPayload);
```

# [Optional parameters](#optional-parameters)

Some blockchains have different technicalities when building transactions, handled using optional parameters.

## [externalTxId](#externaltxid)

The <em>critical</em> practice to avoid multiple identical POST transaction requests being processed more than once is to use the <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">externalTxId</code> parameter in the <a target="_self" href="/reference/createtransaction">Create a new transaction</a> API call.

The <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">externalTxId</code> value is the internal identifier of the transaction you will have on your end.

This value is securely stored in the Fireblocks system and additional transaction requests with the same <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">externalTxId</code> value will not be processed on our system.

<CodeGroup>
```typescript

  assetId: "ETH",
  amount: "0.001",
  externalTxId: "UniqueIdentifier", // the unique identifier of the transaction outside of Fireblocks
  source: {
    type: TransferPeerPathType.VaultAccount,
    id: "0",
  },
  destination: {
    type: TransferPeerPathType.VaultAccount,
    id: "1",
  },
  note: "Your first transaction!",
};

const createTransaction = async (
  transactionPayload: TransactionRequest,
): Promise<CreateTransactionResponse | undefined> => {
  try {
    const transactionResponse = await fireblocks.transactions.createTransaction(

        transactionRequest: transactionPayload,

    console.error(error);
  }
```
