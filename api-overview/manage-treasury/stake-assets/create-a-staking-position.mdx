  # Create a Staking Position

1.  [Approve the terms of service](/reference/approvetermsofservicebyproviderid) of your desired staking provider (Kiln or Figment):

<CodeGroup>
```typescript
const FireblocksSDK = require("fireblocks-sdk").FireblocksSDK;  
const fireblocks = new FireblocksSDK(privateKey, apiKey);  

const approveRequest = await fireblocks.approveStakingProviderTermsOfService("kiln");

```
</CodeGroup>  

2.  [Create your staking position](/reference/stake) :

<CodeGroup>
```typescript
const stakeRequest = await fireblocks.executeStakingStake(
  "SOL",{  
		vaultAccountId:"1",  
		providerId:"kiln",  
		stakeAmount:"100",
		txNote:"stake 100 SOL from vaultId 1, 24 Jan 2024"
  }
);

```
</CodeGroup>

<strong>Important:</strong><br>The response object from the staking action includes an ID that serves as the unique identifier for the staking position.<br>It's essential to save this ID for future reference. If you lose this ID, you can retrieve all the IDs from the staking position in your workspace by using the <a target="_self" href=""><button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">fireblocks.getStakingPositions()</code></a> function.

  
## [Monitor and sign the staking request](#monitor-and-sign-the-staking-request)

<a target="_self" href="/reference/getdelegationbyid">Find the relevant transaction</a> - After creating the staking position, find the new transaction ID using <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">getStakingPosition(id)</code>. The transaction ID can be found under <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">responseBody.relatedTransactions[0].txId</code>:

  
<CodeGroup>
```typescript
const positionId = stakeRequest.id  
const position = await fireblocks.getStakingPosition(positionId);

// When creating a new staking position, there is only 1 transaction  
const txId = position.relatedTransactions[0].txId

// ... sign it

```
</CodeGroup>

To sign the transaction, please follow the necessary steps outlined. Learn more about the [transaction signing request and approval process](https://support.fireblocks.io/hc/en-us/articles/12006018592156-API-Co-Signer-Overview) .

Once the stake position is created, you can monitor the position status and available actions, such as unstaking or withdrawing, using <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">getPosition(positionId)</code>.
