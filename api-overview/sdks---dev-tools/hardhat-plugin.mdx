  # Hardhat Plugin

# [Overview](#overview)

Hardhat is a development environment for Ethereum software. It consists of different components for editing, compiling, debugging and deploying your smart contracts and dApps, all of which work together to create a complete development environment.

# [Using Hardhat](#using-hardhat)
## [Installing Hardhat](#installing-hardhat)

You can skip this section if you already have Hardhat installed or follow that [Hardhat Installation guide](https://hardhat.org/hardhat-runner/docs/getting-started) on the Hardhat website for all details.

We provide you with a very basic process here for convenience:

1.  Install the Hardhat package.

<CodeGroup>
```shell
npm install --save-dev hardhat

```
</CodeGroup>

2.  Initialize a new Hardhat project using the following Hardhat command.  
    This creates a basic project for you to set up directories and configuration files.

<CodeGroup>
```shell
npx hardhat
```

</CodeGroup>

## [Hardhat integration](#hardhat-integration)

The [Fireblocks Hardhat Plugin](https://github.com/fireblocks/hardhat-fireblocks) helps seamlessly integrate Fireblocks into your [Hardhat](https://hardhat.org/) development stack.

You can use it to deploy contracts, sign messages, and send transactions.

### [Installation](#installation)
<CodeGroup>
```shell
npm install @fireblocks/hardhat-fireblocks

```
</CodeGroup>

Import the plugin into your <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">hardhat.config.js</code> or <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">hardhat.config.ts</code> file:

<CodeGroup>
```js
require("@fireblocks/hardhat-fireblocks");
const { ApiBaseUrl } = require("@fireblocks/fireblocks-web3-provider");

```

```js
import "@fireblocks/hardhat-fireblocks";
import { ApiBaseUrl } from "@fireblocks/fireblocks-web3-provider";

```
</CodeGroup>### [Configuration](#configuration)

This plugin extends the <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">HttpNetworkUserConfig</code> object with an optional <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">fireblocks</code> field.

This is an example of how to set this up in your Hardhat configuration file:

<CodeGroup>
```js
module.exports = {
  solidity: "0.8.17",
  networks: {
    sepolia: {
      url: "https://sepolia.drpc.org",
      fireblocks: {
        apiBaseUrl: ApiBaseUrl.Sandbox, // If using a sandbox workspace
        privateKey: process.env.FIREBLOCKS_API_PRIVATE_KEY_PATH,
        apiKey: process.env.FIREBLOCKS_API_KEY,
        vaultAccountIds: process.env.FIREBLOCKS_VAULT_ACCOUNT_IDS,
      }
    },
  },
};

```
</CodeGroup>

If you are not using a Sandbox environment, but rather a regular one, just comment the apiBaseUrl line.

Delete the <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">Lock.sol</code> file that came with your Hardhat boilerplate template:

<CodeGroup>
```shell
rm contracts/Lock.sol

```
</CodeGroup>## [Deploy Contract](#deploy-contract)

Now that you have a Hardhat project set up, create your smart contract <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">contracts/hello.sol</code> in the project folder and then deploy it to the Sepolia Ethereum Testnet.

### [Step 1: Create and compile the Solidity file](#step-1-create-and-compile-the-solidity-file)
<CodeGroup>
```solidity
pragma solidity ^0.8.17;

contract HelloWorld {
    string public greet = "Hello World!";
}

```
</CodeGroup>

Run the following command to compile it:

<CodeGroup>
```shell
npx hardhat compile

```
</CodeGroup>### [Step 2: Update the deployment script](#step-2-update-the-deployment-script)

Replace the contents of the <button aria-label="Copy Code" class="rdmd-code-copy fa"></button><code class="rdmd-code lang- theme-light" data-lang="" name="" tabindex="0">deploy.js</code> script with the following basic contract deployment flow:

<CodeGroup>
```javascript
const hre = require("hardhat");

async function main() {
  const factory = await hre.ethers.getContractFactory("HelloWorld");
  const contract = await factory.deploy();

  await contract.deployed();

  console.log("contract deployed to:", contract.address);
}

// We recommend this pattern to be able to use async/await everywhere
// and properly handle errors.
main().catch((error) => {
  console.error(error);
  process.exitCode = 1;
});

```
</CodeGroup>### [Step 3: Deploy smart contract](#step-3-deploy-smart-contract)

Deploy your contract to the Ethereum Sepolia Testnet:

<CodeGroup>
```shell
npx hardhat run --network sepolia scripts/deploy.js

```
</CodeGroup>

Or:

<CodeGroup>
```shell
HARDHAT_NETWORK=sepolia node scripts/deploy.js

```
</CodeGroup>

This should take a couple of minutes to run if everything was configured correctly.

Once the deploy script has finished running, you will see this message if the contract was deployed successfully:

<CodeGroup>
```shell
contract deployed to: <contract_address>

```
</CodeGroup>
